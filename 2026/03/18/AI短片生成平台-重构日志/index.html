AI短片生成平台-重构日志

发表于2026-03-18|更新于2026-03-18|AI ComicFusion
|浏览量:

写在前面

今天终于把这个项目好好整理了一下。之前写得比较粗糙，很多东西都是模拟实现的，没有真正接入 API。这次算是下了决心，要把它做成一个真正可用的模块化架构。

为什么要重构？

之前的代码虽然能跑，但问题一堆：

  • 前后端混在一起，耦合太高
  • AI API 都是 mock 的，实际没法生成视频
  • 类型定义不完整，很多地方 any 满天飞
  • 扩展性差，想加个新功能都麻烦

正好用户给了一个很详细模块化开发计划，我就按着这个思路重新捋了一遍。

今天的改动

1. 把类型定义搞扎实了

之前 types.ts 已经有一些基础类型，但远远不够。这次扩充了很多：

这些类型就像是工程的「骨架」，有了它们之后，代码写起来会踏实很多。

2. 做了配置模块

新建了 src/lib/config/ai.config.ts，专门管 API 配置相关的事情：

  • 支持切换不同的 AI 提供商（即梦 / Runway / 本地）
  • 从环境变量读取配置
  • 检查 API 密钥有没有配置好
  • 根据质量问题返回不同的参数

这样以后想换 AI 提供商，只需要改个环境变量就行，不用动代码。

3. 写了真正的 API 客户端

之前 generate 接口是模拟的，这次写了两个真正的客户端：

  • jimeng.client.ts - 即梦 AI 的调用封装
  • runway.client.ts - Runway 的调用封装

功能包括：

  • 发起视频生成请求
  • 查询任务状态
  • 轮询等待生成完成（带进度回调）

还做了统一的导出 index.ts，这样前端调用的时候不用关心用的是哪个 AI。

4. 项目管理 API

之前没有完整的项目 CRUD，这次补上了：

接口 功能
GET /api/project 列表（支持分页、排序、筛选）
POST /api/project 创建
GET /api/project/[id] 详情
PUT /api/project/[id] 更新
DELETE /api/project/[id] 删除
GET /api/video/status 查询生成状态

5. 简单的内存数据库

不想一开始就用太重的数据库，整了个内存版的 db.ts，用 Map 存数据。后期想换 PostgreSQL 也很方便。

6. 环境变量配置

写了个 .env.example 和 .env.local，把需要的环境变量都列出来了。后面真正接 API 的时候，按着这个配置就行。

项目结构（更新后）

怎么启用真正的 AI 生成？

目前代码已经写好了，但 API 密钥还没配置。要启用的话：

  1. 复制 .env.local.example 为 .env.local
  2. 填入你的 API 密钥：

然后代码会自动检测有没有配置好，没配置的话会友好地提示。

接下来的计划

  • 把主页面改成真正调用新的 API
  • 接入一个真的 AI 视频生成服务测一下
  • 加上用户系统（注册 / 登录）
  • 配音、TTS 功能
  • 作品库页面

小结

今天的工作主要是把「基础设施」打扎实了。类型、配置、API 客户端、数据库层都有了初步的框架。后面再写功能的时候，会顺畅很多。

构建已经跑通了，没有报错。可以先这样跑一段时间，后面再迭代。

未完待续…
