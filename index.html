目录1 链表 链表数据结构学习笔记：链表（Linked List） 在大学的数据结构课程中，链表（Linked List） 是最经典、最基础的内容之一。它和数组一样，都是用来存储一组数据的，但实现方式完全不同。很多同学刚开始学的时候觉得抽象，其实理解了“节点”和“指针”的概念后，链表就没那么难了。 一、为什么需要链表？在 C/C++ 或 Java 中，数组的长度一旦定义，就不能随意改变。如果我们需要频繁地插入或删除元素，数组的效率就会很低，因为可能要移动一大堆数据。 链表就是为了解决这个问题： 插入/删除更高效：不需要整体移动数据，只要修改几个指针。 动态扩展：不必事先定义大小，想加多少节点都可以。 当然，链表也有缺点： 不能像数组那样直接通过下标访问某个元素，只能从头节点一个一个找下去。 需要额外存储指针，浪费一点空间。 二、链表的基本结构链表由一个个 节点（Node） 组成，每个节点包含两部分： 数据域：存放实际的数据。 指针域：存放下一个节点的地址。 在 C 语言中，可以这样定义一个单链表节点： 1234typedef struct Node {...
