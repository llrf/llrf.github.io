链表

数据结构学习笔记：链表（Linked List）

在大学的数据结构课程中，链表（Linked List） 是最经典、最基础的内容之一。它和数组一样，都是用来存储一组数据的，但实现方式完全不同。很多同学刚开始学的时候觉得抽象，其实理解了“节点”和“指针”的概念后，链表就没那么难了。

一、为什么需要链表？

在 C/C++ 或 Java 中，数组的长度一旦定义，就不能随意改变。如果我们需要频繁地插入或删除元素，数组的效率就会很低，因为可能要移动一大堆数据。

链表就是为了解决这个问题：

插入/删除更高效：不需要整体移动数据，只要修改几个指针。

动态扩展：不必事先定义大小，想加多少节点都可以。

当然，链表也有缺点：

不能像数组那样直接通过下标访问某个元素，只能从头节点一个一个找下去。

需要额外存储指针，浪费一点空间。

二、链表的基本结构

链表由一个个 节点（Node） 组成，每个节点包含两部分：

数据域：存放实际的数据。

指针域：存放下一个节点的地址。

在 C 语言中，可以这样定义一个单链表节点：

比如一个链表存储 [1, 2, 3]，结构是这样的：

[1 | next] -> [2 | next] -> [3 | NULL]

NULL 表示链表的结尾。

三、链表的常见操作

  1. 创建链表

我们先创建几个节点并链接起来：

#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
int data;
struct Node* next;
} Node;

int main() {
// 创建三个节点
Node* head = (Node*)malloc(sizeof(Node));
Node* second = (Node*)malloc(sizeof(Node));
Node* third = (Node*)malloc(sizeof(Node));

head->data = 1;
head->next = second;

second->data = 2;
second->next = third;

third->data = 3;
third->next = NULL; // 链表结束

// 打印链表
Node* p = head;
while (p != NULL) {
    printf("%d -> ", p->data);
    p = p->next;
}
printf("NULL\n");

return 0;

}

运行结果：

1 -> 2 -> 3 -> NULL

  1. 插入节点

假设我们要在 head 后面插入一个新节点 x = 99：

Node* newNode = (Node*)malloc(sizeof(Node));
newNode->data = 99;
newNode->next = head->next; // 新节点指向原来head的下一个
head->next = newNode; // head指向新节点

链表变为：

1 -> 99 -> 2 -> 3 -> NULL

  1. 删除节点

假设要删除 99 这个节点，只需要修改指针：

Node* temp = head->next; // temp指向99
head->next = temp->next; // 跳过99
free(temp); // 释放内存

链表变回：

1 -> 2 -> 3 -> NULL

四、链表的类型

单链表：每个节点只指向下一个节点。

双链表：每个节点有两个指针，分别指向前一个和后一个节点，方便双向遍历。

循环链表：最后一个节点的指针指向头节点，形成一个“环”。

五、总结

链表是数据结构课程里最重要的内容之一，理解链表能帮助我们掌握后续更复杂的结构（比如栈、队列、哈希表）。要掌握链表，最关键的就是：

理解节点与指针的关系。

熟悉基本操作（创建、插入、删除、遍历）。

多画图！链表操作时先在草稿纸上画出节点和指针的变化，思路会更清晰。

学习链表，就像在拼接积木一样，把一个个节点用“指针”串起来。只要多写代码、多调试，链表一定能拿下！
