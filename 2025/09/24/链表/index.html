数据结构学习笔记：单链表专题

一、引言：为什么需要链表？

在数据结构中，线性表有两种主要实现方式：

顺序表（数组）：元素在内存中是连续存放的。
链表：元素分散存放，通过指针连接起来。

顺序表的优点是能快速按下标访问，
但缺点是： 插入、删除需要移动大量元素，效率低。 长度固定，不够灵活。

链表的优点是： 插入、删除效率高，只需修改指针。 长度可动态变化，不受限制。
缺点是： 不能按下标直接访问，需要从头遍历。 多消耗一些存储空间（指针域）。

二、单链表的定义与结构

单链表（Singly Linked List） 是最基础的一种链表，每个节点包含两部分：

1.数据域：存放数据元素。
2.指针域：存放下一个节点的地址。

定义了一个 结构体类型，名字叫 struct Node。
它描述了链表中“节点”的存储方式。

struct Node* next;

节点里的 指针域，类型是“指向 struct Node 的指针”。
它的作用是指向下一个节点，从而把多个节点“链”起来。

typedef … Node;

typedef 给 struct Node 起了一个别名 Node。
这样我们以后就可以写 Node* p;，而不用写冗长的 struct Node* p;。

带头结点 vs 不带头结点

带头结点：第一个节点是“头结点”，不存储实际数据，仅起到统一操作的作用。
不带头结点：第一个节点就是存储数据的节点。

推荐：带头结点，方便操作，尤其是插入和删除。

空链表

带头结点：head->next == NULL
不带头结点：head == NULL

  1. 为什么要在 tail 后面插入

我们想用 尾插法 来建链表。尾插法的特点是：
新节点要接到链表的“最后”，这样链表中节点的顺序就和输入顺序一致。

但是程序要“找到最后一个节点”才能把新节点挂上去。
👉 如果每次都从 head 开始往后找，效率会很低（需要 O(n) 时间）。

所以这里用了一个小技巧：用 tail 始终指向最后一个节点，这样就能 O(1) 时间找到尾巴。

4. 关键点总结

tail 一开始确实是最后的节点（只有头结点的时候）。
每插入一个新节点，就把它挂到 tail 后面，然后再更新 tail。

因为 tail 总是指向最后节点，所以“接在 tail 后面”正好就是“接到链表的末尾”。

✅ 所以：
“把新节点接在 tail 后面”这句话，等价于“把新节点接到当前链表的最后位置”。
更新完 tail 后，它仍然是最新的最后节点。

六、单链表的应用

栈的实现

用头插法实现入栈（push）和删除首节点实现出栈（pop）。

队列的实现

用链表头指针和尾指针，插入在尾，删除在头。

多项式的存储与运算

每一项（系数、指数）存放在一个节点，方便多项式加减操作。

七、总结与对比

特性 顺序表 单链表
存储方式 连续 分散（靠指针连接）
按下标访问 O(1) O(n)
插入删除 O(n) O(1)（已知位置）
空间 固定，可能浪费 动态分配，灵活
应用场景 查找多、修改少 插入删除多

结论：

如果主要是查找 → 顺序表更合适。

如果主要是插入删除 → 单链表更高效。
